Boost Agents Setup Guide

How to request, configure, and deliver an AI agent.

Overview

Boost Agents is an internal platform for building and deploying AI agents powered by Gemini, Claude, or OpenAI. Each agent gets its own URL, a chat interface, and a configuration sidebar for admins.

Agents can connect to Gmail, Google Calendar, and Slack — and can be triggered both from the web UI and from Slack by mentioning the bot. Past conversations are saved and accessible from the history panel.

ℹ
The platform is managed centrally. Creating a new agent is a request you make to Boaz Thaller — you don't need to touch GitHub or GCP yourself.

1. Request an agent

Send the following to Boaz to get a new agent created:

B
Boaz Thaller
boazt@moveoboost.com

Include in your request:

  • Agent name — e.g. Acme HR Assistant
  • Your email — you'll be added as an admin
  • Client emails (if any) — anyone else who needs admin access
  • AI provider preference — Gemini (default), Claude, or OpenAI
  • Which tools to enable — Gmail, Calendar, Slack, Web Fetch, etc.

2. Provide secrets

Boaz will create the GitHub repo and ask you for the following. Some are inherited automatically — you only need to provide the ones marked Required.

3. Deploy (automatic)

Once Boaz adds the secrets to the repo, the deployment runs automatically. You don't need to do anything. The pipeline:

  1. Provisions a new GCP project for this agent
  2. Builds the server and web app
  3. Deploys to Cloud Run — a single URL serves everything

The whole process takes about 5–8 minutes on first deploy, ~2 minutes on subsequent deploys.

Finding your agent URL

After the deploy finishes, the URL appears in two places:

1
GitHub Actions summary (easiest) Open the repo on GitHub → Actions tab → click the latest workflow run → the URL is shown prominently at the top of the summary page under "✦ Agent deployed".
2
GCP Console Go to console.cloud.google.com → select the agent's GCP project (named boost-<agent-name>-v7) → Cloud Run → click the service → the URL is shown at the top.
⚠
If the first run fails, just go to GitHub Actions → click the failed run → Re-run all jobs. A timing issue during GCP provisioning occasionally causes the first run to fail; the second always succeeds.

4. Configure the agent

Open the agent URL and sign in with Google. As an admin you'll see the configuration sidebar on the right.

Model

Each user picks their own preferred model — changes are personal and don't affect other users. Available models depend on which API keys were provided. Admins can click Publish Changes to set the global default.

Instructions

The agent's system prompt — its role, personality, and constraints. Changes are personal by default. Publish to make them the global baseline for all users.

Tools

Admins enable or disable tools globally. Google tools (Gmail, Calendar) activate automatically once a user connects their account — no extra toggle needed.

Connections

Each user connects their own Google account via the Connect link in the sidebar. Once connected, Gmail and Calendar work immediately and stay connected across sessions.

Skills

Knowledge blocks added to the agent's context. Use for SOPs, product docs, FAQs — anything the agent should always know. Enable/disable without redeploying. The agent uses them automatically when relevant.

Automations

Scheduled tasks that run on a cron or a specific one-time datetime. The agent runs the prompt for every connected user at the scheduled time, using their own Gmail/Calendar access. Each automation shows a tag with who created it.

Chat history

All conversations are saved automatically. The history panel on the left lists past chats — click any to resume. A new empty chat opens by default when you load the app.

Publish Changes

Commits the global config to GitHub. Changes to model and instructions take effect immediately in your own session — publishing makes them the permanent default for all users and future deploys.

Delivering to a client

1
Request the agent Send Boaz the agent name, admin emails, and any API keys needed. He'll create the repo and add secrets.
2
Add client users to OAuth For clients to connect Gmail or Calendar, Boaz must add their email to the OAuth test users list. See Adding OAuth clients.
3
Configure the agent Log in, write the system prompt, enable tools, add skills. Click Publish Changes.
4
Send the URL to the client One URL — the client signs in with Google, connects their services, and starts chatting.

Adding OAuth clients

For users to connect their Gmail or Google Calendar to the agent, their email must be added to the OAuth consent screen. Only Boaz can do this.

★
Contact Boaz Thaller at boazt@moveoboost.com with the list of email addresses that need Google service access.

What Boaz does

  1. Go to console.cloud.google.com → select project boost-agents-496211
  2. Navigate to APIs & Services → OAuth consent screen
  3. Scroll to Test users → click + Add users
  4. Enter the client's Gmail address and save
⚠
Maximum 100 test users total across all agents. For larger deployments, contact Boaz to discuss options.

Note: users can still log in to the agent and use it without being added here. The Google service connection (Gmail, Calendar) is the only thing that requires this step.

Slack bot setup

Each agent can have its own Slack bot. Users can ask the agent to send Slack messages from the chat, and can also @mention the bot directly in Slack to trigger the agent — with full access to Gmail, Calendar, and all other tools.

Step 1 — Create a Slack app

  1. Go to api.slack.com/apps → Create New App → From scratch
  2. Name it (e.g. HR Agent) and select your workspace → Create App

Step 2 — Add bot scopes

Under OAuth & Permissions → Bot Token Scopes, add:

ScopeWhy
chat:writePost messages to channels
channels:readList public channels
groups:readList private channels (optional)
users:readLook up Slack users by ID
users:read.emailMatch Slack user to their Gmail/Calendar connection

Step 3 — Install the app

Under OAuth & Permissions → click Install to Workspace. Copy the Bot User OAuth Token (xoxb-...) — this is your SLACK_BOT_TOKEN.

Step 4 — Enable Event Subscriptions

  1. Under Event Subscriptions, toggle Enable Events on
  2. Set Request URL to https://your-agent-url/slack/events — Slack will verify it automatically once your agent is deployed
  3. Under Subscribe to bot events, add app_mention
  4. Save changes and reinstall the app if prompted

Step 5 — Get the Signing Secret

Go to Basic Information → App Credentials → Signing Secret → Show. This is your SLACK_SIGNING_SECRET.

Step 6 — Add repo secrets

Send Boaz the two values so he can add them as repo secrets: SLACK_BOT_TOKEN and SLACK_SIGNING_SECRET.

Step 7 — Enable Slack in the agent sidebar

Log in to the agent → Tools → toggle Slack on → Publish Changes.

Step 8 — Invite the bot to channels

In Slack, open the channel → click the channel name → Integrations → Add an App → search for your bot name.

Using the bot

@HRAgent send an email to john@acme.com with today's agenda
@HRAgent what meetings do I have this afternoon?
@HRAgent send a message to #general saying the standup is at 10am

The bot replies in-thread. It matches your Slack email to your connected Gmail/Calendar, so each person gets personalised access. If you haven't connected via the web UI yet, the bot will prompt you to do so.

ℹ
Each agent has its own Slack app and bot. If you have three agents, you create three Slack apps — each pointing to its own agent URL.

API access

The agent exposes a REST API. Authenticate with the x-api-key header.

Basic request

curl -X POST https://your-agent-url/api/chat \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{"message": "Hello!", "history": []}'

With Gmail / Calendar access

Pass userEmail to tell the agent which connected user's Google account to use:

curl -X POST https://your-agent-url/api/chat \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{"message": "What meetings do I have today?", "history": [], "userEmail": "user@example.com"}'

Response format

{
  "reply": "You have 3 meetings today…",
  "toolUses": [
    { "name": "calendar_list_events", "input": "...", "output": "..." }
  ]
}
